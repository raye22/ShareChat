# Perplexity Chat Scraper - JSON Output Structure

This document describes the structure of the JSON file generated by the Perplexity chat scraper.

## Overview

The scraper captures Perplexity.ai shared conversation pages and outputs a structured JSON file containing the conversation metadata, all turns (user and assistant messages), sources, images, and inline citations.

## Top-Level Structure

```json
{
  "url": "string",
  "turns_count": "integer",
  "metadata": {},
  "turns": []
}
```

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `url` | string | The original URL of the scraped conversation |
| `turns_count` | integer | Total number of conversational turns (user + assistant messages) |
| `metadata` | object | Page metadata including creator, views, and last update info |
| `turns` | array | Array of turn objects representing the conversation flow |

---

## Metadata Object Structure

```json
{
  "created_by": "string",
  "last_updated": "string",
  "views": "string",
  "other_info": "string | null"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `created_by` | string | User ID of the conversation creator |
| `last_updated` | string | Last update date (e.g., "October 20, 2025") |
| `views` | string | Number of views as string (e.g., "23") |
| `shares` | string | Number of shares as string (e.g., "10") |
| `other_info` | string \| null | Additional metadata if available |

---

## Turn Object Structure

Each turn represents either a user message or an assistant (LLM) response.

### Common Fields (All Turns)

```json
{
  "index": 1,
  "role": "user" | "llm",
  "thinking": null,
  "warning": "string | null",
  "analysis": null,
  "plain_text": "string | null",
  "plain_text_md": "string | null",
  "source_bar": null,
  "images_bar": null,
  "answer_bar": null,
  "code": null,
  "aggregated_text": "string | null",
  "links": null,
  "source": null
}
```

| Field | Type | Description |
|-------|------|-------------|
| `index` | integer | Sequential turn number (1-indexed) |
| `role` | string | Either `"user"` or `"llm"` |
| `thinking` | object \| null | Extended thinking block (not supported under perplexity) |
| `warning` | string \| null | Warning/notice text displayed above the message (not supported under perplexity) |
| `analysis` | array \| null | Array of analysis blocks (currently not captured) |
| `plain_text` | string \| null | Plain text version of the message content from Answer tab |
| `plain_text_md` | string \| null | Markdown-formatted version of the message content from Answer tab |
| `source_bar` | object \| null | Sources tab content (assistant only, see below) |
| `images_bar` | object \| null | Images tab content (assistant only, see below) |
| `answer_bar` | object \| null | Complete Answer tab content (assistant only, see below) |
| `code` | array \| null | Array of code blocks (not supported under perplexity) |
| `aggregated_text` | string \| null | Same as the plain_text_md |
| `links` | array \| null | Array of hyperlinks in the message (not supported under perplexity) |
| `source` | array \| null | Array of inline citation sources from Answer tab |

- Only `plain_text` and `aggregated_text` contain content

## Assistant (LLM) Turn Structure

Assistant turns contain tab-specific content (Answer, Sources, Images) and inline citations.

### Answer Bar Object

The `answer_bar` contains the main response content from the Answer tab:

```json
{
  "thinking": null,
  "plain_text": "Here's what I found about quantum doorway states...",
  "plain_text_md": "Here's what I found about **quantum doorway states**...",
  "source": [
    {
      "name": "Phys.org",
      "snippet": "The quantum door mystery: Electrons that can't find the exit",
      "link": "https://phys.org/news/..."
    }
  ],
  "aggregated_text": "Full text with expanded inline citations...",
  "assistant_steps": null
}
```

| Field | Type | Description |
|-------|------|-------------|
| `thinking` | object \| null | Extended thinking content (if available) |
| `plain_text` | string | Plain text of the answer |
| `plain_text_md` | string | Markdown-formatted answer text |
| `source` | array | Inline citation sources referenced in the answer |
| `aggregated_text` | string | Answer text with sources expanded inline |
| `assistant_steps` | object \| null | Step-by-step reasoning (if available) |

### Source Object (Inline Citations)

Each item in the `source` array represents an inline citation:

```json
{
  "name": "Phys.org",
  "snippet": "The quantum door mystery: Electrons that can't find the exit - Phys.org",
  "link": "https://phys.org/news/2025-10-quantum-door-mystery-electrons-exit.html"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Source domain/website name |
| `snippet` | string | Title or snippet text from the source |
| `link` | string | Full URL to the source |

### Source Bar Object

The `source_bar` contains the full Sources tab content:

```json
{
  "meta": {
    "meta_label": "10 web pages",
    "count": 10,
    "icons": ["https://favicon.url1", "https://favicon.url2"],
    "meta_raw": ""
  },
  "title": "Search Results",
  "items": [
    {
      "profile_name": "Phys.org",
      "profile_src": "https://www.google.com/s2/favicons?...",
      "title": "The quantum door mystery: Electrons that can't find the exit - Phys.org",
      "snippet": "In quantum mechanics, electrons can tunnel through barriers...",
      "url": "https://phys.org/news/...",
      "date": "3 days ago"
    }
  ]
}
```

#### Meta Object

| Field | Type | Description |
|-------|------|-------------|
| `meta_label` | string | Summary label (e.g., "10 web pages") |
| `count` | integer | Number of source items |
| `icons` | array | List of unique favicon URLs |
| `meta_raw` | string | Raw metadata text |

#### Source Item Object

| Field | Type | Description |
|-------|------|-------------|
| `profile_name` | string | Source website name |
| `profile_src` | string | Favicon/profile image URL |
| `title` | string | Full title of the source article/page |
| `snippet` | string | Excerpt or description from the source |
| `url` | string | Full URL to the source page |
| `date` | string | Relative date (e.g., "3 days ago") |

### Images Bar Object

The `images_bar` contains the Images tab content:

```json
{
  "meta": {
    "count": 5
  },
  "title": "Images",
  "items": [
    {
      "image_url": "https://cdn.example.com/image.jpg",
      "image_alt": "Quantum mechanics illustration",
      "source_url": "https://nature.com/article",
      "source_name": "nature",
      "source_favicon": "https://www.google.com/s2/favicons?domain=nature.com",
      "width": "319.374"
    }
  ]
}
```

#### Image Item Object

| Field | Type | Description |
|-------|------|-------------|
| `image_url` | string | Direct URL to the image |
| `image_alt` | string | Alt text/description of the image |
| `source_url` | string | URL of the page containing the image |
| `source_name` | string | Domain name of the source |
| `source_favicon` | string | Favicon URL for the source domain |
| `width` | string | Image width in pixels (as string) |

## Usage Notes

### Working with the Data

**Get all user messages:**
```python
user_messages = [turn for turn in data['turns'] if turn['role'] == 'user']
```

**Get all assistant responses with sources:**
```python
responses_with_sources = [
    turn for turn in data['turns'] 
    if turn['role'] == 'llm' and turn.get('source_bar')
]
```

**Extract all inline citations:**
```python
all_citations = []
for turn in data['turns']:
    if turn['role'] == 'llm' and turn.get('source'):
        all_citations.extend(turn['source'])
```

**Get all source URLs from Sources tab:**
```python
all_source_urls = []
for turn in data['turns']:
    if turn.get('source_bar') and turn['source_bar'].get('items'):
        for item in turn['source_bar']['items']:
            all_source_urls.append(item['url'])
```

**Extract all images:**
```python
all_images = []
for turn in data['turns']:
    if turn.get('images_bar') and turn['images_bar'].get('items'):
        all_images.extend(turn['images_bar']['items'])
```

**Get complete answer text with citations:**
```python
for turn in data['turns']:
    if turn['role'] == 'llm' and turn.get('answer_bar'):
        answer = turn['answer_bar']
        print(f"Answer: {answer['plain_text']}")
        print(f"Citations: {len(answer.get('source', []))}")
        print(f"Full text: {answer['aggregated_text']}")
```

### Text Format Recommendations

- Use **`plain_text`** for simple text processing or search
- Use **`plain_text_md`** for rendering in markdown viewers
- Use **`aggregated_text`** when you need the full text with inline citations
- Use **`answer_bar`** to access the complete Answer tab content including all metadata
- Use **`source_bar`** to access full source details with snippets and metadata
- Use **`images_bar`** to access all related images with attribution

### Tab Structure

Perplexity organizes responses into three tabs:

1. **Answer Tab** (`answer_bar`): Main response with inline citations
2. **Sources Tab** (`source_bar`): Detailed list of all sources used
3. **Images Tab** (`images_bar`): Related images with source attribution

Each tab's content is independently captured and stored in the corresponding field.
