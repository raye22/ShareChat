# Gemini Share Scraper - JSON Output Structure

This document describes the structure of the JSON file generated by the Gemini share scraper.

## Overview

The scraper captures Gemini shared conversation pages (gemini.google.com/share/*) and outputs a structured JSON file containing the conversation metadata and all turns (user queries and model responses).

**What makes Gemini unique:** Gemini share pages have a clean, simple structure with clear metadata about the model used, creation time, and publication time. Unlike other AI chat platforms, Gemini focuses on straightforward query-response pairs without complex features like inline citations or code execution.

## Top-Level Structure

```json
{
  "url": "string",
  "model": "string | null",
  "created_at": "string | null",
  "published_at": "string | null",
  "turns_count": "integer",
  "turns": [],
  "meta_raw": "string"
}
```

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `url` | string | The original URL of the scraped conversation |
| `model` | string \| null | Gemini model version used (e.g., "Gemini 1.5 Flash", "Gemini 1.5 Pro") |
| `created_at` | string \| null | When the conversation was created (e.g., "December 15, 2024 at 3:45 PM") |
| `published_at` | string \| null | When the conversation was published/shared (e.g., "December 16, 2024 at 9:30 AM") |
| `turns_count` | integer | Total number of conversational turns |
| `turns` | array | Array of turn objects representing the conversation flow |
| `meta_raw` | string | Raw metadata text extracted from the page |

---

## Metadata Extraction

The scraper extracts metadata from text patterns on the page, typically found in the `.publish-time` element or similar containers. The metadata follows this pattern:

```
Created with [Model Name] [Date] at [Time] Published on [Date] at [Time]
```

Example metadata string:
```
Created with Gemini 1.5 Flash December 15, 2024 at 3:45 PM Published on December 16, 2024 at 9:30 AM
```

The scraper uses regular expressions to parse:
- **Model name**: The Gemini version (e.g., "Gemini 1.5 Flash", "Gemini 1.5 Pro", "Gemini Advanced")
- **Created date and time**: When the user created the conversation
- **Published date and time**: When the conversation was shared publicly

---

## Turn Object Structure

Each turn represents a single exchange in the conversation, containing both the user's query and the model's response.

```json
{
  "index": 1,
  "user_text": "string",
  "model_text": "string",
  "user_time": "string | null",
  "model_time": "string | null"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `index` | integer | Sequential turn number (1-indexed) |
| `user_text` | string | The user's query or prompt |
| `model_text` | string | The model's response |
| `user_time` | string \| null | Timestamp for user message (currently not captured) |
| `model_time` | string \| null | Timestamp for model response (currently not captured) |

**Note:** Unlike Perplexity and Grok, Gemini uses a combined turn structure where each turn contains BOTH the user query and the model response together, rather than alternating separate user/assistant turns.

---

## Turn Extraction Process

The scraper identifies turns using the following CSS selectors:

1. **Primary selector**: `share-turn-viewer` elements within `.chat-history`
2. **Fallback**: `share-turn-viewer` elements anywhere in the `share-viewer` root

For each turn:
- **User query** is extracted from the `user-query` element
- **Model response** is extracted from the `response-container` element

## Usage Notes

### Working with the Data

**Get all user queries:**
```python
user_queries = [turn["user_text"] for turn in data["turns"]]
```

**Get all model responses:**
```python
model_responses = [turn["model_text"] for turn in data["turns"]]
```

**Find conversations by model version:**
```python
flash_conversations = [conv for conv in conversations if conv["model"] and "Flash" in conv["model"]]
pro_conversations = [conv for conv in conversations if conv["model"] and "Pro" in conv["model"]]
```

**Extract conversation metadata:**
```python
conversation_info = {
    "model": data["model"],
    "created": data["created_at"],
    "published": data["published_at"],
    "turns": data["turns_count"]
}
```

**Reconstruct full conversation as dialogue:**
```python
def format_conversation(data):
    lines = []
    lines.append(f"Model: {data['model']}")
    lines.append(f"Created: {data['created_at']}")
    lines.append(f"Published: {data['published_at']}")
    lines.append("")
    
    for turn in data["turns"]:
        lines.append(f"👤 User: {turn['user_text']}")
        lines.append("")
        lines.append(f"🤖 Gemini: {turn['model_text']}")
        lines.append("")
        lines.append("-" * 50)
        lines.append("")
    
    return "\n".join(lines)
```

**Count total words in conversation:**
```python
total_words = sum(
    len(turn["user_text"].split()) + len(turn["model_text"].split())
    for turn in data["turns"]
)
```

**Filter by date range:**
```python
from datetime import datetime

def parse_date(date_str):
    # Parse "December 15, 2024 at 3:45 PM" format
    return datetime.strptime(date_str.split(" at ")[0], "%B %d, %Y")

recent_convos = [
    conv for conv in conversations 
    if conv["created_at"] and parse_date(conv["created_at"]).year == 2024
]
```

**Export to CSV for analysis:**
```python
import csv

def export_turns_to_csv(json_data, output_file):
    with open(output_file, 'w', newline='', encoding='utf-8') as f:
        writer = csv.writer(f)
        writer.writerow(['URL', 'Model', 'Turn Index', 'User Query', 'Model Response'])
        
        for turn in json_data["turns"]:
            writer.writerow([
                json_data["url"],
                json_data["model"],
                turn["index"],
                turn["user_text"],
                turn["model_text"]
            ])
```

## Additional Files Generated

The scraper generates multiple output files for each conversation:

| File | Description |
|------|-------------|
| `{url_id}.json` | Main structured data (this document) |
| `{url_id}.html` | Complete page HTML snapshot |
| `{url_id}.mhtml` | MHTML archive of the page (optional, may not always succeed) |

### Optimized Driver Options

The scraper uses an optimized Chrome driver with:
- Headless mode for faster processing
- Disabled images for speed
- Reduced window size
- Eager page load strategy
- Disabled unnecessary features (extensions, plugins, etc.)
