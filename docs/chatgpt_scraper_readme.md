# ChatGPT Share Scraper - JSON Output Structure

This document describes the structure of the JSON file generated by the ChatGPT share scraper.

## Overview

The scraper captures ChatGPT shared conversation pages (chatgpt.com/share/*) and outputs a structured JSON file containing the conversation metadata, all message exchanges, and embedded script data.

**What makes ChatGPT unique:** ChatGPT's share format includes fine-grained paragraph-level segmentation with automatic language detection for each text block. Each message is broken down into individual paragraphs, and the scraper captures embedded React application state data that contains subscription and account information.

## Top-Level Structure

```json
{
  "num_turns": {
    "user": "integer",
    "assistant": "integer",
    "script": "integer"
  },
  "meta_data": {
    "model_slug": "string",
    "create_time": "string"
  },
  "conversation": []
}
```

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `num_turns` | object | Count of different turn types in the conversation |
| `num_turns.user` | integer | Number of user message paragraphs |
| `num_turns.assistant` | integer | Number of assistant response paragraphs |
| `num_turns.script` | integer | Number of script/metadata blocks |
| `meta_data` | object | High-level conversation metadata |
| `meta_data.model_slug` | string | OpenAI model identifier (e.g., "text-davinci-002-render-sha", "gpt-4", "gpt-3.5-turbo") |
| `meta_data.create_time` | string | Conversation creation timestamp (format: "YYYY-MM-DD HH:MM:SS") |
| `conversation` | array | Array of turn arrays, where each turn can contain multiple paragraph objects |

---

## Conversation Structure

The `conversation` field is a **nested array structure**. Each element in the outer array represents a logical turn (either a user message or an assistant response), and contains an inner array of paragraph objects.

```json
[
  [
    { "author": "user", "type": "paragraph", "content": "...", "lang": [...] }
  ],
  [
    { "author": "assistant", "type": "paragraph", "content": "...", "lang": [...] },
    { "author": "assistant", "type": "paragraph", "content": "...", "lang": [...] }
  ],
  // ... more turns
  [
    { "author": "script", "type": "script", "content": "...", "payload": [...] }
  ]
]
```

### Why Nested Arrays?

Each turn is an array because:
1. **User messages** typically contain 1 paragraph object
2. **Assistant responses** often contain multiple paragraph objects (multi-paragraph answers, lists, etc.)
3. **Script blocks** contain React application state and metadata

## Paragraph Object Structure

### User Paragraph

```json
{
  "author": "user",
  "type": "paragraph",
  "content": "Can you summarize an Excel sheet for me?",
  "lang": [
    ["en", "English", 0.8476262092590332],
    ["ro", "Romanian", 0.011584354564547539],
    ["km", "Khmer", 0.005949575453996658]
  ]
}
```

### Assistant Paragraph

```json
{
  "author": "assistant",
  "type": "paragraph",
  "content": "Certainly! I'd be happy to help you summarize an Excel sheet...",
  "lang": [
    ["en", "English", 0.937307596206665],
    ["id", "Indonesian", 0.0030249019619077444],
    ["ru", "Russian", 0.002691370900720358]
  ]
}
```

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `author` | string | Either `"user"` or `"assistant"` for message paragraphs |
| `type` | string | Always `"paragraph"` for message content |
| `content` | string | The text content of this paragraph |
| `lang` | array | Language detection results (array of [code, name, confidence]) |

---

## Language Detection Array

Each paragraph includes automatic language detection with confidence scores:

```json
"lang": [
  ["en", "English", 0.937307596206665],
  ["id", "Indonesian", 0.0030249019619077444],
  ["ru", "Russian", 0.002691370900720358]
]
```

Structure: `[language_code, language_name, confidence_score]`

- **language_code**: ISO 639-1 two-letter language code (e.g., "en", "es", "zh")
- **language_name**: Full language name in English
- **confidence_score**: Float between 0 and 1 indicating detection confidence
- Results are sorted by confidence (highest first)
- Multiple language predictions indicate multilingual or ambiguous content

---

## Script Object Structure

Script objects contain embedded React application state and metadata:

```json
{
  "author": "script",
  "type": "script",
  "content": "window.__reactRouterContext.streamController.enqueue(\"...\");",
  "payload": [],
  "metadata": {
    "model_slug": "text-davinci-002-render-sha",
    "created_time": "2024-01-08 17:08:19"
  }
}
```

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `author` | string | Always `"script"` for embedded code |
| `type` | string | Always `"script"` |
| `content` | string | Raw JavaScript code string from the React hydration |
| `payload` | array | Parsed structured data from the script content |
| `metadata` | object | Extracted conversation metadata (optional, appears in some script blocks) |

### Script Payload Data

The `payload` field contains various types of data depending on the script block:

- **Account state**: User subscription info, features, settings
- **Conversation metadata**: Creation time, model used
- **Application state**: React Router context, feature flags
- **User profile**: User ID, account type, workspace info

## Usage Notes

### Working with the Data

**Count total user and assistant messages (not paragraphs):**
```python
user_messages = sum(1 for turn in data['conversation'] 
                   if turn and turn[0]['author'] == 'user')
assistant_messages = sum(1 for turn in data['conversation'] 
                        if turn and turn[0]['author'] == 'assistant')
```

**Extract all user queries:**
```python
user_queries = []
for turn in data['conversation']:
    if turn and turn[0]['author'] == 'user':
        # Combine all paragraphs in this user turn
        full_message = ' '.join(p['content'] for p in turn if p['type'] == 'paragraph')
        user_queries.append(full_message)
```

**Extract all assistant responses:**
```python
assistant_responses = []
for turn in data['conversation']:
    if turn and turn[0]['author'] == 'assistant':
        # Combine all paragraphs in this assistant turn
        full_response = '\n\n'.join(p['content'] for p in turn if p['type'] == 'paragraph')
        assistant_responses.append(full_response)
```
