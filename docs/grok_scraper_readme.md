# Grok Chat Scraper - JSON Output Structure

This document describes the structure of the JSON file generated by the Grok chat scraper.

## Overview

The scraper captures Grok.com shared conversation pages and outputs a structured JSON file containing the conversation metadata and all turns (user and assistant messages). 

Grok displays sources in a dedicated right-column pane that contains citations from both web pages and X (Twitter) posts. These sources are embedded directly in the assistant's responses, making Grok particularly valuable for tracking social media and web sources together.

## Top-Level Structure

```json
{
  "url": "string",
  "turns_count": "integer",
  "turns": []
}
```

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `url` | string | The original URL of the scraped conversation |
| `turns_count` | integer | Total number of conversational turns (user + assistant messages) |
| `turns` | array | Array of turn objects representing the conversation flow |

---

## Turn Object Structure

Each turn represents either a user message or an assistant (LLM) response.

### Common Fields (All Turns)

```json
{
  "index": 1,
  "role": "user" | "llm",
  "thinking": null,
  "warning": null,
  "analysis": null,
  "plain_text": "string | null",
  "plain_text_md": "string | null",
  "code": null,
  "aggregated_text": "string | null",
  "links": null,
  "source": null
}
```

| Field | Type | Description |
|-------|------|-------------|
| `index` | integer | Sequential turn number (1-indexed) |
| `role` | string | Either `"user"` or `"llm"` |
| `thinking` | object \| null | Extended thinking block with metadata (if available) |
| `warning` | string \| null | Warning/notice text displayed with the message (if any) |
| `analysis` | array \| null | Array of analysis blocks (currently not captured) |
| `plain_text` | string \| null | Plain text version of the message content |
| `plain_text_md` | string \| null | Markdown-formatted version with links as `[text]` only |
| `code` | array \| null | Array of code blocks (not applicable for Grok) |
| `aggregated_text` | string \| null | Full markdown text with links in `[text](url)` format |
| `links` | array \| null | Array of all hyperlinks extracted from the message |
| `source` | array \| null | Array of source objects from the right-column pane (assistant only) |


**Note:** User turns typically have:
- Most fields = `null` (only used for assistant responses)
- Only `plain_text` and `aggregated_text` contain content

## Assistant (LLM) Turn Structure

Assistant turns contain the main response text, extracted links, and detailed source information from the right-column pane.

### Thinking Block Object

The `thinking` object captures extended thinking content when available:

```json
{
  "meta": {
    "header": "Thinking...",
    "time": "2s"
  },
  "content": "Let me break down the query...",
  "content_md": "Let me **break down** the query..."
}
```

| Field | Type | Description |
|-------|------|-------------|
| `meta.header` | string | Header text (typically "Thinking...") |
| `meta.time` | string | Time spent thinking (e.g., "2s", "1m, 50s") |
| `content` | string | Plain text of thinking content |
| `content_md` | string | Markdown-formatted thinking content |

### Links Array

The `links` array contains all hyperlinks extracted from the response text, with position tracking:

```json
[
  {
    "text": "IBM's quantum roadmap",
    "href": "https://www.ibm.com/quantum/roadmap",
    "start": 145,
    "end": 170
  },
  {
    "text": "Google's breakthrough",
    "href": "https://blog.google/technology/quantum/",
    "start": 245,
    "end": 268
  }
]
```

| Field | Type | Description |
|-------|------|-------------|
| `text` | string | The visible link text |
| `href` | string | The full URL |
| `start` | integer | Start position in `plain_text_md` string |
| `end` | integer | End position in `plain_text_md` string |

**How links are captured:**
- The scraper extracts all markdown links `[text](url)` from the HTML
- Each link's position is tracked in the `plain_text_md` version (where links appear as `[text]` without URLs)
- The `aggregated_text` field preserves the full `[text](url)` format for easy reading
- Links are preserved in document order, including duplicates if the same link appears multiple times

#### Source Meta Object

| Field | Type | Description |
|-------|------|-------------|
| `meta_label` | string | Summary label from the source chip (e.g., "15 web pages", "8 𝕏 posts") |
| `count` | integer | Number of source items (parsed from meta_label) |
| `icons` | array | List of favicon/avatar URLs from the source chip |
| `meta_raw` | string | Raw text from the source chip |

#### Source Item Object

Each item in the `items` array represents a single source (web page or X post):

| Field | Type | Description |
|-------|------|-------------|
| `url` | string | Full URL to the source |
| `author` | string | Author/publisher name (e.g., "Nature Journal", "IBM Quantum") |
| `handle` | string | X/Twitter handle (e.g., "@IBMQuantum"), empty for web sources |
| `date` | string | Relative date (e.g., "3 days ago", "2h ago") |
| `profile_alt` | string | Alt text for profile image |
| `profile_src` | string | URL to profile image/favicon |
| `snippet` | string | Excerpt or preview text from the source |
| `raw_text` | string | Complete raw text from the source card |

#### Source Markdown Fields

| Field | Type | Description |
|-------|------|-------------|
| `md_full` | string | Full markdown-formatted content with `[text](url)` links |
| `md_clean` | string | Markdown content with URLs removed (links as `[text]` only) |
| `ok` | boolean | Whether the source was successfully parsed |
| `error` | string \| null | Error message if parsing failed |

## Usage Notes

### Working with the Data

**Get all user messages:**
```python
user_messages = [turn for turn in data['turns'] if turn['role'] == 'user']
```

**Get all assistant responses with sources:**
```python
responses_with_sources = [
    turn for turn in data['turns'] 
    if turn['role'] == 'llm' and turn.get('source')
]
```

**Extract all web sources:**
```python
web_sources = []
for turn in data['turns']:
    if turn['role'] == 'llm' and turn.get('source'):
        for source_group in turn['source']:
            # Filter for web pages (no Twitter handle)
            if 'web' in source_group['meta']['meta_label'].lower():
                for item in source_group['items']:
                    web_sources.append(item)
```

**Extract all X (Twitter) posts:**
```python
x_posts = []
for turn in data['turns']:
    if turn['role'] == 'llm' and turn.get('source'):
        for source_group in turn['source']:
            # Filter for X posts
            if '𝕏' in source_group['meta']['meta_label'] or 'posts' in source_group['title'].lower():
                for item in source_group['items']:
                    x_posts.append(item)
```

**Extract all links from responses:**
```python
all_links = []
for turn in data['turns']:
    if turn['role'] == 'llm' and turn.get('links'):
        all_links.extend(turn['links'])
```

**Get aggregated text (with full markdown links):**
```python
for turn in data['turns']:
    if turn['role'] == 'llm':
        # This includes full [text](url) format
        print(turn['aggregated_text'])
```

**Reconstruct source attribution:**
```python
for turn in data['turns']:
    if turn['role'] == 'llm' and turn.get('source'):
        print(f"Response: {turn['plain_text']}\n")
        print("Sources:")
        for source_group in turn['source']:
            print(f"\n{source_group['title']} ({source_group['meta']['meta_label']}):")
            for item in source_group['items']:
                print(f"  - {item['author']}: {item['url']}")
```

### Text Format Recommendations

- Use **`plain_text`** for simple text processing without any formatting
- Use **`plain_text_md`** for markdown with links appearing as `[text]` (no URLs)
- Use **`aggregated_text`** when you need full markdown with working `[text](url)` links
- Use **`links`** array to programmatically access all hyperlinks with their positions
- Use **`source`** array to access detailed source metadata from the right-column pane

### Understanding Grok's Source Structure

Grok organizes sources into groups that appear as chips on the left side:
- **Web sources**: Labeled as "X web pages" - contains articles, papers, websites
- **X (Twitter) sources**: Labeled as "X 𝕏 posts" - contains tweets/posts from X/Twitter

When a user clicks on a source chip:
1. The right-column pane opens
2. A title appears at the top (e.g., "Web Results", "Posts from 𝕏")
3. Individual cards display with detailed metadata
4. Each card includes: author, date, snippet, profile image, and clickable URL

The scraper captures:
- Pre-click metadata (count, label, icons)
- Post-click content (title, individual items, full markdown)
- Both web and X sources in a unified structure

## Additional Files Generated

The scraper also generates:

| File | Description |
|------|-------------|

  Fail to capture the view times data on the top right, but the information may be contained in the html file.
